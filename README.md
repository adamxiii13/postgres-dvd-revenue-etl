# PostgreSQL Automated ETL Pipeline: DVD Rental Revenue Analysis

## Project Overview
This project demonstrates an automated ETL (Extract, Transform, Load) pipeline built entirely within PostgreSQL. It is designed to analyze DVD rental data and generate a monthly summary report calculating the total revenue for each film category (genre).

## Business Value
The insights generated by this automated report allow business stakeholders, specifically the Ordering Department, to:
* **Optimize Inventory:** Identify high-demand film categories to ensure adequate stock levels.
* **Identify Weak Points:** Determine which genres are underperforming.
* **Analyze Seasonality:** Review historical monthly data to identify seasonal demand patterns (e.g., increased demand for Romance films in February).

## Technical Architecture & Skills Demonstrated
This pipeline utilizes several advanced SQL features to ensure data integrity, format consistency, and hands-off automation:
* **Custom User-Defined Functions (UDFs):** Developed functions like `dollar_fmt` to enforce standardized financial formatting for shareholder readability, and `rental_month` to extract and cast timestamp data for seasonal grouping.
* **Database Triggers:** Implemented a trigger (`refresh_summary_trigger`) that automatically aggregates total revenue into a summary table the moment a new transaction is inserted into the detailed records.
* **Stored Procedures:** Created a procedure (`refresh_rental_reports`) that handles the core ETL process—truncating old data, extracting from the normalized database, joining across 6 disparate tables, and loading the transformed data into the reporting tables.
* **Job Scheduling:** Designed to be automated using **pgAgent**, scheduling the stored procedure to execute at 12:01 AM on the first day of every month.

## Repository Structure
* `01_table_creation.sql`: DDL for detailed and summary reporting tables.
* `02_custom_functions.sql`: UDFs for data transformation (currency formatting, date extraction).
* `03_triggers.sql`: Trigger function and execution statement for real-time aggregation.
* `04_stored_procedure.sql`: The primary ETL procedure combining data from `payment`, `rental`, `inventory`, `film`, `film_category`, and `category` tables.
